# PriceTracker 全面升级文档化流程计划

## 概述
按照WORKFLOW.md中的Agent流水线流程，为PriceTracker项目的全面升级（UI重构 + 架构重构 + 新功能）产出完整的规划文档。

## 目标
通过完整的PM → Research → UI/UX → Architect流程，产出：
1. `docs/requirements.md` - 需求文档
2. `docs/research.md` - 调研报告
3. `docs/design-identity.md` - 气质定位（确认后产出`docs/design.md`完整设计规范）
4. `docs/architecture.md` - 技术方案

## 项目现状分析

### 当前功能
- 价格总览：商品聚合视图，支持搜索/筛选/排序
- 购买记录：时间线视图，支持多维度筛选
- 数据分析：4个统计卡片 + 4个图表
- AI智能导入：基于LLM的收据识别导入
- CRUD完整：添加/编辑/删除购买记录和商品
- 导入导出：JSON/CSV格式
- 暗色/亮色主题自动切换
- 响应式设计（480px/768px/1024px断点）

### 当前技术栈
- 纯前端：HTML + CSS + JavaScript（零依赖）
- 存储：localStorage
- 图表：手写SVG甜甜圈图 + CSS柱状图
- 架构：全局对象模式（window.Store/Utils/Charts/App）

### 当前问题
- `app.js` 1104行，超过WORKFLOW.md规定的300行限制
- 使用onclick内联处理器，不符合规范
- 模块间直接引用全局对象，未使用EventBus
- 无测试套件
- localStorage版本管理仅做全量重置，无迁移机制

## 执行流程

### 阶段1：PM Agent - 需求文档
**目标**：明确全面升级的需求范围和优先级
**输出**：`docs/requirements.md`
**内容**：
- 一句话定义
- 核心使用场景
- 核心功能（按优先级排序）
- 不做的东西（明确排除）
- 技术约束
- 验收标准

**关键决策点**：
- 新功能范围：多用户支持？云端同步？数据可视化增强？
- UI重构程度：保持现有功能仅美化，还是重新设计交互？
- 架构重构范围：仅代码拆分，还是引入构建工具/框架？

### 阶段2：Research Agent - 调研报告
**目标**：分析现有方案，确定技术路线
**输出**：`docs/research.md`
**内容**：
- 现有方案对比（3-5个竞品/参考项目）
- 推荐方案（自己做/用现成的/基于xx二次开发）
- 如果自己做，参考哪些项目

**调研方向**：
- GitHub开源价格追踪/个人财务项目
- 现代前端架构方案（模块化、状态管理）
- 数据可视化库选型
- 移动端PWA方案

### 阶段3：UI/UX Agent - 设计规范
**目标**：定义项目的视觉语言和交互规范
**输出**：
1. `docs/design-identity.md` - 气质定位（先确认）
2. `docs/design.md` - 完整设计规范（确认后输出）

**内容**：
- 气质定位：参考对象、整体感受、视觉记忆点
- 配色方案
- 字体规范
- 间距系统
- 页面布局
- 关键交互
- 响应式策略

### 阶段4：Architect Agent - 技术方案
**目标**：设计可维护、可扩展的技术架构
**输出**：`docs/architecture.md`
**内容**：
- 技术栈选型
- 文件结构设计
- 数据结构设计
- 模块接口定义
- 迁移策略

**关键决策点**：
- 是否引入构建工具（Vite/Webpack）
- 是否使用前端框架（Vue/React）
- 模块化方案（ES Modules vs 全局对象）
- 状态管理方案（EventBus vs 集中式状态）

## 执行规则
1. 每个阶段完成后停下来等待用户确认
2. 不自动进入下一阶段
3. 用户确认后才产出下一阶段文档
4. 每个阶段可多次迭代，直到用户满意

## 验证方法
1. 每个文档产出后，检查是否符合WORKFLOW.md中的输出格式要求
2. 确保文档内容与项目现状和升级目标一致
3. 用户确认后，方可进入下一阶段

**B. 添加showItemHistory()函数（在closeEditItemModal()后）**
```javascript
showItemHistory(itemId) {
  const item = Store.getItemById(itemId);
  if (!item) return;

  const modal = document.getElementById('modal-item-history');
  
  // 标题
  document.getElementById('history-title').textContent = `📊 ${item.name} - 购买历史`;
  
  // 摘要统计
  document.getElementById('history-category').textContent = item.category;
  document.getElementById('history-count').textContent = `${item.totalPurchases}次`;
  document.getElementById('history-price-range').textContent = 
    `${Utils.formatPrice(item.minPrice)} ~ ${Utils.formatPrice(item.maxPrice)}`;
  document.getElementById('history-avg-price').textContent = Utils.formatPrice(item.avgPrice);
  
  // 按日期排序购买记录（最新在前）
  const purchases = [...item.purchases].sort((a, b) => new Date(b.date) - new Date(a.date));
  
  // 构建表格行
  const tbody = document.getElementById('history-tbody');
  tbody.innerHTML = purchases.map((p, i) => {
    // 价格趋势（与前一次购买比较）
    let trendHtml = '';
    if (i < purchases.length - 1) {
      const prevPrice = purchases[i + 1].price;
      const diff = p.price - prevPrice;
      if (diff < -0.01) {
        trendHtml = '<span class="price-trend down">↓</span>';
      } else if (diff > 0.01) {
        trendHtml = '<span class="price-trend up">↑</span>';
      } else {
        trendHtml = '<span class="price-trend flat">→</span>';
      }
    }
    
    // 促销徽章
    const promoHtml = p.isPromo 
      ? `<span class="promo-badge">🏷️ ${Utils.escapeHtml(p.promoType || '促销')}</span>`
      : '-';
    
    // 规格信息
    const specHtml = p.specQty && p.specUnit 
      ? `${p.specQty}${Utils.escapeHtml(p.specUnit)}` 
      : '-';
    
    return `
      <tr>
        <td>${p.date}</td>
        <td class="price">${Utils.formatPrice(p.price)}${trendHtml}</td>
        <td>${p.quantity}</td>
        <td class="price">${Utils.formatPrice(p.totalPrice)}</td>
        <td>${Utils.escapeHtml(p.channel)}</td>
        <td>${specHtml}</td>
        <td>${promoHtml}</td>
        <td>${p.notes ? Utils.escapeHtml(p.notes) : '-'}</td>
      </tr>
    `;
  }).join('');
  
  modal.classList.add('show');
},
```

### 3. 功能特点
1. **可点击商品名称**：在总览表格中，商品名称显示为可点击链接样式
2. **摘要统计**：显示分类、总购买次数、价格区间、平均价格
3. **购买历史表格**：按时间倒序显示所有购买记录
4. **价格趋势指示器**：显示每次购买相比前一次的价格变化（↑↓→）
5. **促销信息**：促销购买显示徽章
6. **响应式设计**：表格可滚动，支持大量记录
7. **一致性**：遵循现有模态框模式，支持Escape键和点击背景关闭

### 4. 验证方法
1. 启动应用（使用Live Server或直接打开index.html）
2. 在总览页面点击任意商品名称
3. 验证模态框显示正确的商品信息
4. 检查购买历史表格是否完整
5. 测试价格趋势指示器是否正确
6. 测试关闭模态框（Escape键、点击背景、关闭按钮）
7. 测试边界情况（单次购买、多次购买、有促销的购买）

### 5. 注意事项
- 无需修改`store.js`，`Store.getItemById()`已返回完整购买记录数组
- 无需修改`closeAllModals()`，它已通过`querySelectorAll('.modal-overlay.show')`处理所有模态框
- 所有用户数据使用`Utils.escapeHtml()`防止XSS攻击
- 表格使用`max-height: 400px`和`overflow-y: auto`支持大量记录滚动
